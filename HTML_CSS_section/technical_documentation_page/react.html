Introduction to React

React is a JavaScript library, which is often associated with Angular and Vue as the three big JavaScript Frontend frameworks, despite React technically not being a framework at all. React is also split into React Native and React.js, the former being the mobile development version, while the latter is the web development version.

React is built up of components, which is one of the most important parts to learn about React. A stateful component usually has this as the first lines of code in the class:

constructor(props) {
 super(props)
 this.state = {
  name: "",
  age: 18
 }
}

State is what React revolves around, but you're never allowed to redeclare a stateful object. Therefore there is commonly used function in React that reassigns a stateful object with a new value. That function is setState, and is written like so:

this.setState({ name: "John", age: 25 })

Another central part of React is that each component renders into the DOM. That means that every component has to return something that can be rendered to the DOM, but not every component needs to render something. The render method usually looks somewhat like this:

render() { return(<div></ div> ) }

Looking at the example above, we can see a div element being returned in the render method. That is because each component can only return one thing. However, we can pass along as many things as we'd like inside of that div, e.g.:

<div>
<h1>Hello World</h1>
<p>This is a hello world example.</p>
<article>Some text</article>
</div>

That means that if we were to try to return several elements, like the ones above, without wrapping them in a single wrapper element, we'd get an error. However, the wrapper element doesn't have to be a div, it could be anything that can contain other elements. Anyway, this would not work, despite it looking the same in this HTML document:

<h1>Hello World</h1>
<p>This is a hello world example.</p>
<article>Some text</article>
History of React

React was created by Jordan Walke, a software engineer at Facebook. He was influenced by XHP, an HTML component framework for PHP. It was first deployed on Facebook's newsfeed in 2011 and later on Instagram.com in 2012.

The Upsides of Using React

One of the greatest upsides of React is that it is client-side rendered. That means that the user loads the site once, and after that every interaction with the site is really smooth, since all the content of the site is already loaded. React is also great for developers. The component built system makes it easy to structure code, but it also allows for extremely easy refactoring of code, since you can simply refactor a whole component. That makes React loved by both users and developers.

The Downsides of Using React

Everything isn't roses though, because there are some downsides with React. One of the downsides is managing state in a complex application. Because of that complexity, the library 'Redux' is often used alongside React to make state management easier, but React is slowly making progress towards making state management easier without Redux. If you're really doing your job well as a developer, handling state without Redux should very much be possible for you.

Another huge downside of React is actually one of its greatest upside: being client-side rendered. Facebook is ironically a prime example of why it can get bad. Facebook is actually still using React, but as Facebook has grown, aware users may have noticed that the site has become significantly slower to load over the years. That is because Facebook has grown a lot and as the site is growing, the users also has to load more data on start than before, making the initial rendering of the site a slow process.
